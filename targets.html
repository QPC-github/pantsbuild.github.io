Targets Reference¶

This page documents targets available as part of the pants build system.

annotation_processor Module¶

class twitter.pants.targets.annotation_processor.AnnotationProcessor(name, sources, provides=None, dependencies=None, excludes=None, resources=None, processors=None, exclusives=None)[source]¶

Produces a Java library containing one or more annotation processors.

Parameters:
  • name (string) – The name of this target, which combined with this build file defines the target twitter.pants.base.address.Address.
  • sources (list of strings) – A list of filenames representing the source code this library is compiled from.
  • provides (Artifact) – The twitter.pants.targets.artifact.Artifact to publish that represents this target outside the repo.
  • dependencies (list of targets) – List of twitter.pants.base.target.Target instances this target depends on.
  • excludes – List of twitter.pants.targets.exclude.Exclude instances to filter this target’s transitive dependencies against.
  • resources – An optional list of file paths (DEPRECATED) or resources targets (which in turn point to file paths). The paths indicate text file resources to place in this module’s jar.
  • processors – A list of the fully qualified class names of the annotation processors this library exports.
  • exclusives – An optional map of exclusives tags. See CheckExclusives for details.

class twitter.pants.targets.jar_dependency.JarDependency(org, name, rev=None, force=False, ext=None, url=None, apidocs=None, type_=None, classifier=None, mutable=None, exclusives=None)[source]¶

A pre-built Maven repository dependency.

Parameters:
  • org (string) – The Maven groupId of this dependency.
  • name (string) – The Maven artifactId of this dependency.
  • rev (string) – The Maven version of this dependency. If unspecified the latest available version is used.

  • classifier (string) – Classifier specifying the artifact variant to use. Use with_artifact to include multiple artifacts with different classifiers.
  • mutable (boolean) – Inhibit caching of this mutable artifact. A common use is for Maven -SNAPSHOT style artifacts in an active development/integration cycle.

jvm_binary Module¶

class twitter.pants.targets.jvm_binary.Bundle(base=None, mapper=None, relative_to=None)[source]¶

A set of files to include in an application bundle.

Looking for Java-style resources accessible via the Class.getResource API? Those are resources.

Files added to the bundle will be included when bundling an application target. By default relative paths are preserved. For example, to include config and scripts directories:

bundles=[
  bundle().add(rglobs('config/*', 'scripts/*')),
]

To include files relative to some path component use the relative_to parameter. The following places the contents of common/config in a config directory in the bundle.

bundles=[
  bundle(relative_to='common').add(globs('common/config/*'))
]
Parameters:
  • mapper – Function that takes a path string and returns a path string. Takes a path in the source tree, returns a path to use in the resulting bundle. By default, an identity mapper.
  • relative_to (string) – Set up a simple mapping from source path to bundle path. E.g., relative_to='common' removes that prefix from all files in the application bundle.
add(*filesets)[source]¶

Add files to the bundle, where filesets is a filename, globs, or rglobs. Note this is a variable length param and may be specified any number of times.

class twitter.pants.targets.jvm_binary.JvmApp(name, binary, bundles, basename=None)[source]¶

A JVM-based application consisting of a binary plus “extra files”.

Invoking the bundle goal on one of these targets creates a self-contained artifact suitable for deployment on some other machine. The artifact contains the executable jar, its dependencies, and extra files like config files, startup scripts, etc.

Parameters:
  • name (string) – The name of this target, which combined with this build file defines the target twitter.pants.base.address.Address.
  • binary – The twitter.pants.targets.jvm_binary.JvmBinary, or a twitter.pants.targets.pants_target.Pants pointer to one.
  • bundles – One or more twitter.pants.targets.jvm_binary.Bundle‘s describing “extra files” that should be included with this app (e.g.: config files, startup scripts).
  • basename (string) – Name of this application, if different from the name. Pants uses this in the bundle goal to name the distribution artifact. In most cases this parameter is not necessary.
class twitter.pants.targets.jvm_binary.JvmBinary(name, main=None, basename=None, source=None, resources=None, dependencies=None, excludes=None, deploy_excludes=None, configurations=None, exclusives=None)[source]¶

Produces a JVM binary optionally identifying a launcher main class.

Below are a summary of how key goals affect targets of this type:

  • bundle - Creates a self-contained directory with the binary and all its dependencies, optionally archived, suitable for deployment.
  • binary - Create an executable jar of the binary. On the JVM this means the jar has a manifest specifying the main class.
  • run - Executes the main class of this binary locally.
Parameters:
  • name (string) – The name of this target, which combined with this build file defines the target twitter.pants.base.address.Address.
  • main (string) – The name of the main class, e.g., 'com.twitter.common.examples.pingpong.Main'. This class may be present as the source of this target or depended-upon library.
  • basename (string) – Base name for the generated .jar file, e.g., 'pingpong'. (By default, uses name param)
  • source (string) – Name of one .java or .scala file (a good place for a main).
  • resources – List of resources to include in bundle.
  • dependencies – List of targets (probably java_library and scala_library targets) to “link” in.
  • excludes – List of excludes to filter this target’s transitive dependencies against.
  • deploy_excludes – List of excludes to apply at deploy time. If you, for example, deploy a java servlet that has one version of servlet.jar onto a Tomcat environment that provides another version, they might conflict. deploy_excludes gives you a way to build your code but exclude the conflicting jar when deploying.
  • configurations (tuple of strings) – Ivy configurations to resolve for this target. This parameter is not intended for general use.

sources Module¶

class twitter.pants.targets.sources.SourceRoot[source]¶

Allows registration of a source root for a set of targets.

A source root is the base path sources for a particular language are found relative to. Generally compilers or interpreters for the source will expect sources relative to a base path and a source root allows calculation of the correct relative paths.

E.g., a Java compiler probably expects to find .java files for package com.twitter.common.net in *something*/com/twitter/common/net. The source_root command specifies that something.

It is illegal to have nested source roots.
