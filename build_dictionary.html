Pants BUILD Dictionary

A BUILD file defines one or more build targets. A target represents a set of source files, dependencies or other 'nouns' that build commands can act on. The type of a target determines what actions apply to it. For example, a jvm_binary target can be compiled and run.

      The python_requirements macro generates python_requirement_library targets and makes them depend on a _python_requirements_file() target, so that pantsd knows to invalidate correctly when the requirements.txt file changes. We don't want to use a regular files() target for this, as we don't want to consider the requirements.txt a source for the purpose of building pexes (e.g., we don't want whitespace changes to requirements.txt to invalidate the sources pex).

      Deploy jars are executable jars with fully self-contained classpaths and as such, assembling them presents problems given jar semantics.
